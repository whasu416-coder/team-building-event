현재 “숙소 주변 편의시설” 섹션의 추상 약도를 실제 지도 기반 약도 UI로 바꿔줘.

목표:
- 숙소 주변 편의시설을 실제 지도 기반으로 보여준다.
- 추상적인 원형 약도 대신, 공식 지도 API 또는 공식 지도 링크를 활용한 지도형 UI로 만든다.
- 사용자가 각 시설을 누르면 바로 지도 앱에서 위치 확인 또는 검색 결과 확인을 할 수 있게 한다.
- 모바일에서도 보기 편해야 한다.
- 실제 거리, 이동시간, 업체명, 좌표를 임의로 작성하지 않는다.

기준 위치:
- 숙소 기준점은 “군산수송제일오투그란데1단지아파트 306동”으로 한다.
- 기존 숙소 위치 카카오맵 링크는 유지한다.
- 숙소 위치 링크:
https://map.kakao.com/?urlX=435165.0000030022&urlY=684921.9999999995&urlLevel=3&itemId=26707549&q=%EA%B5%B0%EC%82%B0%EC%88%98%EC%86%A1%EC%A0%9C%EC%9D%BC%EC%98%A4%ED%88%AC%EA%B7%B8%EB%9E%80%EB%8D%B01%EB%8B%A8%EC%A7%80%EC%95%84%ED%8C%8C%ED%8A%B8%20306%EB%8F%99&srcId=26707549&map_type=TYPE_MAP

중요:
- 카카오맵 링크의 urlX, urlY 값을 Kakao Maps JavaScript API의 위도·경도로 직접 사용하지 말 것.
- API 지도를 구현할 경우 주소 “군산수송제일오투그란데1단지아파트 306동”을 기준으로 geocoder를 사용하거나, 실제 WGS84 위도·경도를 확인한 뒤 중심 좌표로 사용할 것.
- 좌표 확인이 불가능하면 실제 지도 마커를 임의 배치하지 말고 fallback 카드형 링크 UI로 처리할 것.
- 실제 업체명과 좌표를 확인할 수 없는 경우 특정 상호명, 실제 거리, 도보 시간, 차량 이동 시간을 만들어내지 말 것.

구현 방식:
1. 지도는 실제 지도 기반으로 구현한다.
2. 우선순위는 다음과 같다.
   - 1순위: Kakao Maps JavaScript API 사용
   - 2순위: Google Maps Embed 또는 Google Maps 링크 사용
   - 3순위: API 키가 없거나 좌표 확인이 불가능하면 지도 영역에는 “지도에서 주변 편의시설 보기” 버튼형 fallback UI를 제공하고, 각 항목은 카카오맵 검색 링크로 연결한다.
3. 지도 이미지를 무단 캡처해서 배경 이미지로 넣지 않는다.
4. 공식 API 또는 공식 링크 방식만 사용한다.
5. API 키가 없거나 공개 저장소 배포 가능성이 있으면 키를 코드에 직접 하드코딩하지 않는다.
6. API 키가 필요한 경우 코드 상단에 다음 주석을 넣는다.
   “// TODO: Kakao Maps JavaScript API 키 입력 필요”
7. API 키 사용 시 도메인 제한 또는 환경변수 사용이 가능하도록 주석으로 안내한다.
8. API 키가 없어도 페이지가 깨지지 않도록 fallback UI를 반드시 만든다.

지도 API 키 처리:
- Kakao Maps API 키가 필요한 경우 코드 상단에 다음처럼 표시한다.
  “// TODO: Kakao Maps JavaScript API 키 입력 필요”
- 공개 저장소에 API 키가 노출되지 않도록 주의한다.
- 가능한 경우 환경변수 또는 배포 서비스의 secret/environment setting을 사용하도록 주석을 추가한다.
- API 키가 없을 때도 페이지가 깨지지 않도록 fallback UI를 만든다.
- fallback UI는 실제 지도 대신 큰 버튼 1개와 편의시설 검색 카드 목록을 보여준다.
- fallback 버튼 문구:
  “카카오맵에서 숙소 주변 편의시설 보기”
- fallback 버튼 링크:
  https://map.kakao.com/?q=군산수송제일오투그란데1단지아파트%20306동%20근처%20편의시설

지도 영역 디자인:
- 섹션 제목: “숙소 주변 편의시설”
- 설명 문구: “숙소를 기준으로 도착 후 바로 필요한 편의시설을 지도에서 확인할 수 있습니다.”
- 지도 카드 안에 실제 지도 영역을 넣는다.
- 지도 중심은 숙소로 설정한다.
- 숙소 마커를 가장 강조한다.
- 주변 편의시설은 실제 검색 결과 또는 실제 좌표를 확인할 수 있는 경우에만 다른 색상 마커로 표시한다.
- 실제 좌표를 확인할 수 없는 경우 지도 위에 임의 마커를 찍지 않는다.
- 지도 하단에는 다음 문구를 넣는다.
  “※ 실제 영업시간과 경로는 지도 앱에서 최종 확인해 주세요.”

편의시설 검색 기준:
- 실제 장소명이 확정되지 않은 경우 특정 업체명을 임의로 만들지 않는다.
- 대신 숙소 기준 카카오맵 검색 링크를 사용한다.
- 카드에는 “지도 기준 확인 필요”라고 표시한다.
- 사용자가 버튼을 누르면 해당 카테고리 검색 결과가 열리게 한다.

편의시설 마커 처리:
- 실제 업체명과 좌표를 API 검색 결과로 확인할 수 있는 경우에만 지도 위에 마커를 표시한다.
- 실제 좌표를 확인할 수 없는 경우 지도 위에 임의 마커를 찍지 않는다.
- 이 경우 오른쪽 카드 목록에는 “지도 기준 확인 필요”로 표시하고, 각 카드는 카카오맵 검색 링크로 연결한다.
- 카테고리명은 “대형마트”, “동네마트”, “약국”, “편의점”으로 표기한다.
- 마커를 표시할 수 있는 경우 다음 번호 체계를 사용한다.
